How It Works

Every room in the overworld can become a small platforming challenge. Orange rooms are unclaimed. Explore, choose an unclaimed room, build a platforming challenge and publish it so everyone else can play it!

Build And Publish
  • Drafts stay private until you publish.
  • Published rooms appear in the overworld and both players and builders earn points from challenges being completed.
  • Anyone can edit anyone else's room!
  • The original claimer of a room can revert the room to any previously published state.
  • To prevent people from editing a room you claimed, you can mint it as an NFT for .01e.
Challenges

Right now, the game supports both single-room challenges and short published courses that stitch 2-4 adjacent rooms together. Rooms still handle speedruns, checkpoint sprints, collectible hunts, survival, and other small self-contained tests, while courses let you chain authored room challenges into a longer path.

Install App

On mobile, you can launch WAMP without the normal browser address bar and tabs by installing it to your home screen.

  • Safari: use Share -> Add to Home Screen.
  • Brave / Chrome: open the browser menu and choose Add to Home screen or Install app.
  • Result: launching from the installed icon opens the game in a standalone app shell.

Add Your Own Assets

If you want to contribute art, send it over. This is the short public intake guide; we still review assets by hand and clean them up as needed so they fit the game style, animation flow, and collision rules.

  • Terrain: use a 16x16 grid. Terrain sheets should snap cleanly to that grid and usually come as a PNG sheet.
  • Objects and props: the easiest sizes are 16x16, 16x32, 32x32, 32x48, 48x48, or 64x32.
  • Enemies and hazards: same general pipeline as objects. Simple strips on a fixed frame grid are easiest for patrol, flying, bounce, and animated hazard behavior.
  • Backgrounds: current parallax sets are usually layered PNGs at 576x324 or 960x480. Separate layers are better than one flattened image.
  • Player sprites: these are the fussiest. Our current player workflow starts from a 96x84 source canvas and then gets packed into movement, combat, weapons, and FX atlases.
  • Best format: transparent PNGs, nearest-neighbor / no blur, and consistent pixel scale. If something needs unusual timing, collision, or slicing, we usually configure that by hand during review.
